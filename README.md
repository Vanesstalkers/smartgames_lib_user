# lib/user

Пользователь системы: учётная запись, сессии, кэш в Redis.

## Структура

### Ядро

- **Class.js** — класс `User`: наследует `lib.store.class` с broadcast. Создание пользователя (login, password, token, gender), хэширование пароля, загрузка из хранилища. Кэш в Redis (`addUserToCache`, `updateUserCache`) для распределённой привязки пользователя к воркеру/порту.
- **Session.js** — сессия пользователя (связь с браузером, токен, gameId и т.д.).

### API (`api/`)

- **initSession.js** — инициализация сессии (вход по токену/логину-паролю).
- **update.js** — обновление данных пользователя.
- **generateAvatar.js** — генерация аватара.
- **logout.js** — выход из аккаунта (инвалидация сессий).

## Связи

- Используется в лобби и в игре: `lib.game.User` расширяет контекст пользователя при участии в игре (joinGame, playerId, viewerId).
- Сессии пользователя получают события от игры/лобби через подписки на каналы store.
